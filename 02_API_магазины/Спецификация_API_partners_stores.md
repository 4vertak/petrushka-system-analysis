# Спецификация API эндпоинта
Пример реализации проектирования спецификации REST API запроса при переходе пользователя на новый экран (макет)

Спецификация в формате OpenAPI: [openapi.yaml](./openapi.yaml)

## **Название метода:** GET /api/v1/partners/stores/delivery-slots

**Описание:** Возвращает список доступных магазинов-партнёров с ближайшими слотами доставки для указанного местоположения.

**Метод:** GET

**Заголовки:** 
> - *Authorization: Bearer <user_token>*
> - *Accept: application/json*

**Параметры запроса (query params):**

| Параметр     | Тип          | Обязательный     | Описание     |
|--------------|--------------|------------------|--------------|
| latitude     | number       | нет (опционально) | Широта       |
| longitude    | number       | нет (опционально) | Долгота      |

>**Если не передан, используется основной адрес пользователя из профиля*.

### Коды ответов

**Успешный ответ:** 200 OK (пустой массив stores, если ничего не найдено)

**Коды ошибок:**
- 400 Bad Request - Некорректные параметры запроса (если координаты невалидны),

- 401 Unauthorized - Неавторизованный доступ (если токен просрочен или отсутствует),

- 500 Internal Server Error - Внутренняя ошибка сервера.


### **Примеры запросов:**

1. Предположим, что пользователь уже авторизован и его основной адрес сохранён. Запрос может быть таким (без явной передачи адреса, он определяется сервером по сессии):
```text
GET /api/v1/partners/stores/delivery-slots
Host: api.petrushka.ru
Authorization: Bearer <token>
Accept: application/json
```

2. Если нужно передавать геопозицию, то добавляются query-параметры:
```text
GET /api/v1/partners/stores/delivery-slots?latitude=55.7558&longitude=37.6173
```

### **Структура ответа:** (таблица полей)

| Поле         | Тип          | Описание         |
|--------------|--------------|------------------|
| stores | array | Массив объектов магазинов |
| stores[].storeId | string | Уникальный идентификатор магазина |
| stores[].name | string | Название магазина |
| stores[].deliveryText | string | Готовая строка с информацией о доставке для отображения |
| stores[].externalLink | string | URL для перехода на внешний ресурс магазина |
